libpomerol API

Most of pomerol's API is defined in Pomerol namespace with an exception of the MPI tools that are defined in pMPI.

The entire library is available via a single include file pomerol.hpp.

A pomerol-based exact diagonalization program would normally proceed as follows.

  • Define an expression corresponding to the Hamiltonian of the system either in terms of libcommute's creation/annihilation operators (these are injected into the Pomerol namespace in pomerol/Operators.hpp) or by using lattice presets.
  • Construct an IndexClassification object that carries information about all fermionic single-particle states of the system (typically, by calling MakeIndexClassification()).
  • Construct system's Hilbert space by calling MakeHilbertSpace().
  • Employ a StatesClassification object to reveal invariant subspaces (sectors) of the Hamiltonian.
  • Define, prepare and diagonalize the Hamiltonian matrix of the system.
  • Define, prepare and compute the many-body DensityMatrix of the system.
  • Use a FieldOperatorContainer to allocate matrices of fermionic creation/annihilation operators (CreationOperator / AnnihilationOperator). and transform them into the eigenbasis of the Hamiltonian.
  • Finally, compute some of the following physically relevant quantities.
    • Gibbs ensemble averages of operators of physical observables (EnsembleAverage).
    • Single-particle fermionic Green's functions (GreensFunction, GFContainer).
    • Dynamical susceptibilities – correlators of two MonomialOperator's (Susceptibility).
    • 3-point susceptibilities – correlators of two fermionic and one bosonic quadratic operator (ThreePointSusceptibility).
    • Two-particle fermionic Green's functions and irreducible vertices (TwoParticleGF, TwoParticleGFContainer, Vertex4).

All available classes and functions are grouped into a few modules, so you can check out the Modules page to get an overview of libpomerol's API.
